API endpoint URL for Keystone (also known as Identity service).
  • The administrator for your OpenStack deployment should be able to provide this.
  • If you have an "openrc" file then this is the value of OS_AUTH_URL.
  • If you have a user-friendly web front end, you may find this information under an "API" sub-heading.
    e.g. The "Horizon" Openstack Dashboard has this information under Compute -> Access & Security -> API Access -> Identity.
For example, an Openstack deployment using "v2 authentication" might have an endpoint of https://some.host.name:5000/v2.0
Similarly, an Openstack deployment using "v3 authentication" might have an endpoint of https://some.other.host:5000/v3
